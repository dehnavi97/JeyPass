# صندوق جي‌باس الآمن 🇸🇦

![JeyPass Screenshot](https://raw.githubusercontent.com/dehnavi97/jeyPass/main/screenshot/ar.png)

**JeyPass** هو مدير كلمات مرور حديث وآمن وقادر على العمل دون اتصال بالإنترنت، مصمم بالبساطة والأمان في الاعتبار. يستخدم تشفيرًا قويًا (AES-GCM) لحماية بياناتك، مما يضمن أنك وحدك من يمكنه الوصول إلى بيانات اعتمادك باستخدام كلمة مرور رئيسية. تم بناؤه باستخدام Next.js و Electron و Capacitor، مما يوفر تجربة سلسة عبر منصات **Windows و macOS و Linux و Android و iOS**.

**مستودع GitHub:** [https://github.com/dehnavi97/jeyPass](https://github.com/dehnavi97/jeyPass)

---

## ✨ الميزات الرئيسية

- **يعمل دون اتصال أولاً:** يعمل بشكل كامل دون اتصال بالإنترنت. يتم تخزين بياناتك محليًا وبشكل آمن على جهازك.
- **تشفير قوي:** يستخدم AES-256-GCM لتشفير خزنتك بالكامل. لا يمكن قراءة بياناتك بدون كلمة المرور الرئيسية.
- **متعدد المنصات:**
    - **سطح المكتب (Windows, macOS, Linux):** يعمل كتطبيق سطح مكتب أصلي باستخدام Electron.
    - **الجوال (Android, iOS):** يعمل كتطبيق جوال أصلي باستخدام Capacitor.
- **واجهة مستخدم حديثة:**
  - واجهة مستخدم نظيفة وبديهية وقابلة للتخصيص مع قائمة إجراءات عائمة.
  - **السمات:** سمات فاتحة، داكنة، محيط، صحراء، عقيق وغابة لتخصيص تجربتك.
  - شريط عنوان مخصص قابل للسحب واستمرارية حالة النافذة لتطبيق سطح المكتب.
- **دعم TOTP:** إنشاء وعرض كلمات المرور لمرة واحدة المستندة إلى الوقت (رموز المصادقة الثنائية) لحساباتك.
- **مولد كلمات مرور قوية:** أنشئ كلمات مرور قوية وعشوائية وقابلة للتخصيص.
- **مشاركة آمنة:** شارك بيانات الاعتماد بأمان عبر رمز الاستجابة السريعة (QR).
- **النسخ الاحتياطي والاستعادة:** يمكنك بسهولة نسخ خزنتك المشفرة احتياطيًا واستعادتها عند الحاجة، مع دعم عبر المنصات (سطح المكتب والجوال).
- **مؤقت القفل التلقائي:** مؤقت قفل تلقائي قابل للإيقاف المؤقت لتعزيز الأمان.
- **دعم متعدد اللغات:** متوفر بالكامل باللغات الإنجليزية والفارسية والعربية والفرنسية والتركية والألمانية.

## 🛠️ التقنيات المستخدمة

- **إطار العمل:** Next.js 14 (App Router)
- **سطح المكتب:** Electron
- **الجوال:** Capacitor
- **اللغة:** TypeScript
- **التصميم:** Tailwind CSS
- **مكونات واجهة المستخدم:** ShadCN UI
- **التدويل:** i18next
- **التشفير:** Web Crypto API (AES-256-GCM)

## 🚀 البدء

اتبع هذه الخطوات لإعداد المشروع للتطوير المحلي.

### 1. المتطلبات الأساسية

- [Node.js](https://nodejs.org/) (الإصدار 18 أو أحدث)
- npm أو yarn
- لتطوير Android: [Android Studio](https://developer.android.com/studio)
- لتطوير iOS: [Xcode](https://developer.apple.com/xcode/) و [CocoaPods](https://cocoapods.org/)

### 2. استنساخ المستودع

```bash
git clone https://github.com/dehnavi97/jeyPass.git
cd jeyPass
```

### 3. تثبيت التبعيات

```bash
npm install
```

### 4. التشغيل في وضع التطوير

- **لـ Electron (سطح المكتب):**
  لتشغيل التطبيق في وضع التطوير مع التحديث التلقائي:
  ```bash
  npm run dev:electron
  ```

- **لـ Android:**
  1. مزامنة بنية الويب الخاصة بك مع مشروع Android:
     ```bash
     npm run sync:android
     ```
  2. افتح مشروع Android في Android Studio:
     ```bash
     npm run open:android
     ```
  3. في Android Studio، قم بتشغيل التطبيق على محاكي أو جهاز متصل.

- **لـ iOS:**
  1. أضف منصة iOS (إذا لم تكن موجودة):
     ```bash
     npx cap add ios
     ```
  2. مزامنة بنية الويب مع مشروع iOS:
     ```bash
     npm run sync:ios
     ```
  3. افتح مشروع iOS في Xcode:
     ```bash
     npm run open:ios
     ```
  4. في Xcode، قم بتشغيل التطبيق على محاكي أو جهاز iPhone متصل.

## 📦 خروجی گرفتن برای تولید

### 1. تحضير الأيقونات (خطوة أولى مهمة)

1.  أنشئ أيقونة تطبيقك بدقة **1024x1024** واحفظها كـ `assets/icon.png`.
2.  (اختياري) يمكنك إنشاء أيقونة للوضع المظلم باسم `assets/icon-dark.png` وخلفية شاشة البداية باسم `assets/splash.png`.
3.  قم بتشغيل الأمر التالي لإنشاء جميع أحجام الأيقونات وشاشات البداية تلقائيًا:
    ```bash
    npm run assets:generate
    ```

### 2. خروجی گرفتن برای سطح المكتب (Electron)

- **لـ Windows:** (يجب تشغيله على Windows)
  ```bash
  npm run build:electron
  ```
  سيكون المثبت (`.exe`) في دليل `release`.

- **لـ macOS:** (يجب تشغيله على macOS)
  ```bash
  npm run build:electron
  ```
  سيكون المثبت (`.dmg`) في دليل `release`.

- **لـ Linux:**
  ```bash
  npm run build:electron
  ```
  سيكون المثبت (`.AppImage`) في دليل `release`.

### 3. خروجی گرفتن لـ Android (Capacitor)

1.  **مزامنة المشروع:**
    ```bash
    npm run sync:android
    ```
2.  **الفتح في Android Studio:**
    ```bash
    npm run open:android
    ```
3.  **إنشاء APK موقع:**
    - في Android Studio، انتقل إلى `Build > Generate Signed Bundle / APK...`.
    - حدد **APK**، ثم اتبع التعليمات التي تظهر على الشاشة لإنشاء مخزن مفاتيح وتوقيع تطبيقك.
    - بعد اكتمال البناء، يمكنك العثور على ملف `app-release.apk` في `android/app/release`.

### 4. خروجی گرفتن لـ iOS (Capacitor)

1.  **مزامنة المشروع:**
    ```bash
    npm run sync:ios
    ```
2.  **الفتح في Xcode:**
    ```bash
    npm run open:ios
    ```
3.  **أرشفة وتوزيع التطبيق:**
    - في Xcode، اختر `Any iOS Device (arm64)` كهدف.
    - انتقل إلى `Product > Archive`.
    - بعد اكتمال الأرشفة، ستفتح نافذة المنظم. يمكنك توزيع تطبيقك (للاختبار أو App Store) من هناك.

### 5. خروجی گرفتن كتطبيق ويب تقدمي (PWA)

المشروع مُعدّ بالفعل كـ PWA. بعد تشغيل `npm run build`، تكون جميع الملفات اللازمة في دليل `dist`. يمكنك استضافة هذا الدليل على أي خادم ويب ثابت (مثل Vercel، Netlify، أو GitHub Pages) لتوفير التطبيق كتطبيق ويب.
